摘要：记忆是Agent的基石

虽然大语言模型（LLM）能力强大，但它们本质上是“健忘”的。为了实现通用人工智能（AGI），Agent必须具备长期推理、持续适应和有效交互的能力，这一切都依赖于记忆。当前的Agent记忆研究非常碎片化，术语混乱。本文提出了一个全新的分类体系，从形式（Forms）、功能（Functions）和动态（Dynamics）三个维度重新定义了Agent记忆系统。

  • 统一分类法： 提出了 Forms-Functions-Dynamics 三角分类模型，解决了领域内定义混乱的问题。
  • 系统性综述： 详细梳理了Token级、参数化、潜在记忆三种形式，以及事实、经验、工作记忆三种功能。
  • 未来展望： 指出了从检索式记忆向生成式记忆转变，以及RL（强化学习）驱动的自动化记忆管理的趋势。

背景与概念辨析

什么是Agent记忆？

Agent记忆不仅仅是数据库。它是使Agent能够从过去的交互中学习、保持行为一致性并在复杂环境中进行长期规划的认知基质。

❌ 传统的 RAG

主要关注从静态知识库中检索信息以回答单次查询。它是只读的，不随交互进化。

✅ Agent Memory

关注在多轮交互中维护一个持久的、自我进化的认知状态。它会记录经验、更新事实，并改变未来的行为。

记忆形式：存储介质是什么？

这回答了“记忆存在哪里”的问题。论文将记忆的物理载体分为三类：

1. Token-level Memory

以自然语言或离散符号形式存储在外部数据库中。最常见、可读性最强。

扁平 (Flat): 列表、对话日志 (MemGPT)。
平面 (Planar): 知识图谱、树状结构。
层级 (Hierarchical): 多层摘要金字塔 (GraphRAG)。

2. Parametric Memory

将知识内化到模型权重中。类似于人类的“本能”或“肌肉记忆”。

内部参数: 通过微调(SFT)直接修改模型。
外部参数: 使用LoRA或Adapter模块存储特定记忆。

3. Latent Memory

以连续向量或KV Cache形式存在。机器原生语言，信息密度极高。

生成式: 压缩成 summary vectors。
复用式: 直接保存 KV Cache 状态。

记忆功能：用来做什么？

这回答了“为什么需要记忆”的问题。论文超越了简单的“长期/短期”二分法，提出了更细粒度的功能分类。

事实记忆 (Factual)

"Agent知道什么？"

  • 用户事实: 用户的喜好、身份、历史承诺。保证对话的一致性。
  • 环境事实: 世界知识、文档状态。保证对世界的认知准确。
  • 类比：陈述性记忆

经验记忆 (Experiential)

"Agent如何改进？"

  • 案例 (Cases): 过去的成功/失败轨迹。
  • 策略 (Strategies): 从经验中提炼的高级工作流。
  • 技能 (Skills): 可执行的代码或API调用序列。
  • 类比：程序性记忆

工作记忆 (Working)

"Agent正在思考什么？"

  • 任务内管理: 在单次任务中维护上下文。
  • 上下文折叠: 压缩长历史，保留关键状态。
  • 认知规划: 当前的行动计划和草稿。
  • 类比：大脑的工作台

动态机制：记忆如何运转？

1

记忆形成 (Formation)

如何从原始数据中提取有价值的信息？

  • 语义摘要: 将长文本压缩成概要 (如 MemGPT)。
  • 知识蒸馏: 提取特定的事实三元组或经验规则。
  • 结构化构建: 建立知识图谱或层级树。
2

记忆演化 (Evolution)

如何维护记忆库，使其保持准确和高效？

  • 合并 (Consolidation): 将碎片信息融合成一般性见解。
  • 更新 (Updating): 解决新旧信息冲突，修正错误。
  • 遗忘 (Forgetting): 基于时间或重要性修剪冗余信息。

前沿趋势与未来挑战

从“检索”到“生成”

传统的RAG是“查找并粘贴”。未来的Agent将采用生成式记忆(Generative Memory)，即Agent不只是检索原始片段，而是根据当前上下文动态合成或重构出最相关的记忆表示。这更像人类的记忆重建过程。

RL驱动的记忆管理

目前的记忆规则（如什么该存、什么该忘）多是人工设计的启发式规则。未来将通过强化学习(Reinforcement Learning)让Agent自己学会管理记忆，形成完全端到端优化的记忆策略 (Figure 11)。

多模态与多Agent共享

Agent不仅处理文本，还要处理视觉、音频。此外，多Agent协作需要高效的共享记忆(Shared Memory)机制，实现集体智能，同时解决读写冲突和隐私问题。

可信记忆 (Trustworthy)

Agent记忆可能泄露隐私或产生幻觉。未来的系统必须具备可解释性（知道为什么记住了这个）和安全性（可遗忘权、访问控制）。

结论

"记忆不是辅助性的存储机制，而是Agent实现时间一致性、持续适应和长程能力的基质。"

本文通过统一的分类法（形式、功能、动态）整理了碎片化的研究领域。我们正处于一个转折点：从静态的LLM转向具备自我进化能力的Agent。记忆系统的设计将决定这一转变的成败。未来的方向是完全可学习、自组织、多模态的记忆系统。
