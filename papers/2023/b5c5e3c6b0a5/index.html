摘要

人类行为的可信代理可以赋能从沉浸式环境到人际沟通排练空间再到原型设计工具的各种交互应用。本文介绍了生成式智能体：模拟可信人类行为的计算软件智能体。

生成式智能体醒来、做早餐、去上班；艺术家绘画，作家写作；他们形成观点、注意到彼此并开始对话；他们记住并反思过去的日子，同时规划下一天。

为了支持生成式智能体，作者描述了一种架构，该架构扩展了大语言模型，使用自然语言存储智能体经验的完整记录，随时间将这些记忆合成为更高层次的反思，并动态检索它们以规划行为。

作者在受《模拟人生》启发的交互沙盒环境中实例化生成式智能体，最终用户可以使用自然语言与25个智能体的小镇进行交互。评估表明，这些生成式智能体产生可信的个体和涌现社交行为。

背景与动机

研究背景

四十多年来，研究人员和实践者一直设想能够作为人类行为可信代理的计算智能体。从《模拟人生》等沙盒游戏到认知模型和虚拟环境应用，这些人类行为模拟可以为虚拟空间和社区填充真实的社会现象。

然而，人类行为的空间广阔而复杂。尽管大语言模型在模拟单时间点的人类行为方面取得了显著进展，但确保长期一致性的完全通用智能体需要能够管理不断增长记忆的架构。

研究动机

作者认为，大语言模型为重新审视创建可信智能体的问题提供了机会，前提是能够设计有效的架构将记忆合成为可信行为。本文朝着这样的架构迈出了一步。

成功需要一种能够检索长时间内的相关事件和交互，反思这些记忆以进行概括和推断，并将推理应用于创建在当下和长期行为弧中有意义的计划和反应的方法。

问题与挑战

长期一致性挑战

现有方法难以确保智能体在长时间尺度上的行为一致性，无法处理随着时间推移出现和消失的新交互、冲突和事件。

记忆管理挑战

智能体需要能够检索长时间内的相关事件和交互，反思这些记忆以进行概括和推断，并将推理应用于创建在当下和长期行为弧中有意义的计划和反应。

社交动态挑战

处理多个智能体之间展开的级联社交动态，确保智能体之间的交互产生可信的群体行为，如信息传播、关系形成和协调活动。

架构设计挑战

设计能够存储、合成和应用相关记忆以生成可信行为的智能体架构，同时克服大语言模型有限上下文窗口的限制。

技术挑战细节

上下文窗口限制

大语言模型的有限上下文窗口无法容纳智能体的完整记忆流

记忆检索

需要有效机制从大量记忆中检索最相关信息

行为一致性

确保智能体行为在长时间尺度上保持一致和可信

社交协调

多个智能体之间的复杂社交互动和协调

设计与实现

生成式智能体架构概览

生成式智能体架构的核心是记忆流，这是一个维护智能体经验全面记录的数据库。从记忆流中，记录被检索为相关，以规划智能体的行为并对环境做出适当反应。记录被递归合成为越来越高级的反思，指导行为。一切都以自然语言描述记录和推理，使架构能够利用大语言模型。

技术细节：生成式智能体架构
技术解释：
  • 记忆流: 长期记忆模块，以自然语言记录智能体的观察、反思和计划
  • 检索函数: 结合新近性、重要性和相关性来获取相关记忆，使用加权组合计算分数
  • 反思: 定期生成，当最新事件的重要性分数总和超过阈值（150）时触发
  • 规划: 采用自上而下的递归分解方法，从高级计划逐步细化为具体行动
  • 反应机制: 智能体在每个时间步感知环境，决定继续现有计划或做出反应
设计实现：
实现方法:

架构使用ChatGPT (gpt3.5-turbo)作为基础语言模型。记忆流存储所有观察、反思和计划，检索函数基于新近性（指数衰减，衰减因子0.995）、重要性（语言模型直接评分）和相关性（余弦相似度）选择最相关的记忆。

技术选择:

选择使用自然语言表示所有内容，以便充分利用大语言模型的能力。检索函数使用加权组合计算分数：score = recency + importance + relevance，所有权重设置为1。

优化策略:

使用缓存机制存储智能体摘要描述，定期更新。计划采用递归分解方法，从高级计划逐步细化为具体行动。对话生成采用迭代方式，每个智能体基于记忆和对话历史生成回应。

记忆与检索

记忆流维护智能体经验的全面记录。它是一个记忆对象列表，每个对象包含自然语言描述、创建时间戳和最近访问时间戳。最基本的元素是观察，即智能体直接感知的事件。

新近性

最近访问的记忆对象获得更高分数，使用指数衰减函数（衰减因子0.995）

重要性

通过直接询问语言模型输出1-10的整数分数来区分平凡记忆与核心记忆

相关性

基于与当前情况的语义相似性，使用嵌入向量和余弦相似度计算

检索函数实现

score = α_recency · recency + α_importance · importance + α_relevance · relevance
其中 α_recency = α_importance = α_relevance = 1
所有分数使用min-max缩放归一化到[0,1]范围

反思机制

反思是第二种类型的内存，是智能体生成的更高级、更抽象的思想。它们与观察一起包含在检索过程中。反思会定期生成；在实现中，当智能体感知的最新事件的重要性分数总和超过阈值（150）时生成反思。在实践中，智能体大约每天反思两到三次。

反思过程

步骤1: 确定反思内容 - 使用智能体记忆流中最近的100条记录，提示语言模型生成3个最显著的高级问题
步骤2: 为每个问题收集相关记忆
步骤3: 提示语言模型提取见解并引用作为证据的特定记录
步骤4: 将生成的反思存储在记忆流中，包括指向被引用记忆对象的指针

反思明确允许智能体不仅反思观察，还反思其他反思。因此，智能体生成反思树：树的叶节点表示基本观察，非叶节点表示思想，越往树的高层越抽象和高级。

规划与反应

规划描述智能体未来的行动序列，有助于保持智能体行为随时间的一致性。计划包括位置、开始时间和持续时间。与反思一样，计划存储在记忆流中，并包含在检索过程中。

规划过程采用自上而下的方法，然后递归生成更多细节。第一步是创建概述一天议程的计划。然后智能体保存这个计划，并递归分解它以创建更细粒度的行动。

规划分解过程

高级计划: 创建一天议程的粗略草图，分为5-8个时间块
小时级分解: 将每个时间块分解为小时级行动
分钟级分解: 进一步分解为5-15分钟的详细行动

反应与对话机制

生成式智能体在行动循环中操作，在每个时间步，它们感知周围世界，这些感知观察存储在记忆流中。然后提示语言模型决定智能体是应该继续现有计划还是做出反应。对于对话，基于彼此的记忆生成回应，直到一方决定结束对话。

技术细节：控制评估方法
评估方法：
  • 参与者: 100名美国英语母语者，通过Prolific平台招募
  • 评估方法: 对25个智能体进行两天的模拟后采访，采用受试者内设计
  • 问题类别: 5类（自我知识、记忆、规划、反应、反思），每类5个问题
  • 评分系统: 使用TrueSkill评分系统对条件的可信度进行排名
  • 消融条件: 包括无观察/无反思/无规划；无反思/无规划；无反思；以及完整架构
统计分析：
统计测试:
  • Kruskal-Wallis检验: H(4) = 150.29, p < 0.001
  • Dunn事后检验: 所有条件间差异显著 (p < 0.001)
  • 效应大小: 完全消融 vs 完整架构 d = 8.16
关键发现:
  • 完整架构产生最可信行为
  • 每个架构组件都对性能有重要贡献
  • 反思对于经验合成至关重要
  • 人类众包条件与完全消融条件无显著差异

评估发现

  • 完整架构产生最可信行为（TrueSkill: 29.89）
  • 每个架构组件都对性能有贡献
  • 反思对于经验合成至关重要
  • 智能体能够回忆过去经验，但有时会添加修饰
  • 与先前工作（完全消融）相比，效应大小为8.16个标准差

局限性

  • 智能体有时无法检索相关记忆
  • 偶尔会对记忆进行修饰或产生幻觉（1.3%的响应）
  • 从语言模型继承了过于正式的言语或行为
  • 某些情况下会检索不完整的记忆片段

边界条件和错误

  • 位置选择问题: 随着智能体了解更多位置，可能选择不太典型的位置执行行动
  • 物理规范误解: 难以传达某些位置的物理规范（如单人浴室）
  • 指令调优影响: 智能体行为可能过于礼貌和合作
  • 时间理解: 可能不理解某些地方在特定时间关闭

最终结论

生成式智能体展示了计算智能体模拟可信人类行为的潜力。通过将大语言模型与记忆流、反思和规划机制相结合，作者创建了能够产生可信个体和群体行为的智能体。这项工作为人机交互、社交计算和社会科学的新应用开辟了道路，同时也强调了在部署此类技术时需要考虑的重要伦理和社会影响。
