摘要

这篇论文研究了分布式系统中事件排序的基本问题。作者指出，在分布式系统中，"一个事件在另一个事件之前发生"的概念需要重新审视，因为它只定义了事件的部分排序关系。

核心问题：在分布式系统中，由于缺乏全局时钟和消息传输延迟，无法总是确定两个事件发生的先后顺序。

背景与动机

时间概念是我们思维方式的基础，它源于事件发生顺序这一更基本的概念。在集中式系统中，我们可以依赖物理时钟来确定事件顺序。但在分布式系统中，情况变得复杂。

分布式系统定义

分布式系统由一组空间上分离的进程组成，这些进程通过交换消息进行通信。如ARPA网络就是一个分布式系统。单个计算机也可以被视为分布式系统，其中中央控制单元、内存单元和输入输出通道是独立的进程。

在分布式系统中，有时无法确定两个事件哪个先发生。"发生在之前"关系因此只是系统中事件的部分排序。作者发现，由于人们没有充分意识到这一事实及其含义，经常会出现问题。

集中式系统特点
  • 全局共享时钟

分布式系统挑战

  • 事件顺序不确定

"发生在之前"关系

作者定义了"发生在之前"关系(→)，这是分布式系统中事件排序的基础：

定义：发生在之前关系(→)
  1. 如果a和b是同一进程中的事件，且a在b之前发生，则a→b
  2. 如果a是一个进程发送消息的事件，b是另一个进程接收同一消息的事件，则a→b
  3. 如果a→b且b→c，则a→c

如果a→b且b→a都不成立，则称事件a和b是并发的。

设计与实现

逻辑时钟

为了解决事件排序问题，作者引入了逻辑时钟的概念。逻辑时钟不是物理时钟，而是为事件分配数字的一种方式，这些数字表示事件发生的"逻辑时间"。

时钟条件

对于任何事件a,b：如果a→b，则C(a) < C(b)

这个条件确保逻辑时钟尊重"发生在之前"的因果关系。

为了满足时钟条件，需要满足以下两个实现规则：

规则 IR1

每个进程Pᵢ在任何两个连续事件之间递增Cᵢ

确保同一进程内的事件满足时钟条件

规则 IR2

消息包含时间戳，接收进程将时钟设置为大于当前值和消息时间戳的值

确保跨进程的事件满足时钟条件

全序排序

使用满足时钟条件的时钟系统，可以对所有系统事件进行全序排序。作者定义关系⇒如下：

如果Cᵢ(a) < Cⱼ(b)，或者Cᵢ(a) = Cⱼ(b)且Pᵢ < Pⱼ，则a ⇒ b

其中<是进程的任意全序关系，用于打破时间戳相同的情况。

互斥问题解决方案

作者展示了如何使用全序排序解决分布式系统中的互斥问题。算法需要满足三个条件：

条件 I

已获得资源的进程必须在将其授予另一个进程之前释放它

条件 II

不同的资源请求必须按照它们被发出的顺序被授予

条件 III

如果每个被授予资源的进程最终都释放它，那么每个请求最终都会被授予

算法基于五个规则实现分布式互斥：

  1. 请求资源：进程向所有其他进程发送时间戳请求消息，并将该消息放入自己的请求队列
  2. 接收请求：进程将接收到的请求放入自己的请求队列，并向发送方发送确认
  3. 释放资源：进程从请求队列中移除自己的请求，并向所有其他进程发送释放消息
  4. 接收释放：进程从请求队列中移除相应的请求
  5. 授予资源：当进程的请求在队列中最靠前且已收到所有其他进程的时间戳晚于该请求的消息时，授予资源

物理时钟同步

为了避免异常行为，作者引入了物理时钟同步。物理时钟需要满足两个条件：

条件 PC1

时钟以近似正确的速率运行：|dCᵢ(t)/dt - 1| < κ

κ ≪ 1，对于典型晶体控制时钟，κ ≤ 10⁻⁶

条件 PC2

时钟保持同步：|Cᵢ(t) - Cⱼ(t)| < ε

ε是足够小的常数

物理时钟同步算法基于修改后的IR1'和IR2'规则：

物理时钟同步规则
  • IR1': 如果Pᵢ在物理时间t没有接收消息，则Cᵢ在t可微且dCᵢ(t)/dt > 0
  • IR2': 发送消息时包含时间戳；接收消息时将时钟设置为max(Cᵢ(t'-0), Tₘ + μₘ)

时钟同步定理

假设一个强连通进程图，直径为d，始终遵守规则IR1'和IR2'。假设对于任何消息m，μₘ ≤ μ（对于某个常数μ），并且对于所有t ≥ t₀：

  1. PC1成立
  2. 存在常数τ和ξ，使得每τ秒通过每条弧发送一个不可预测延迟小于ξ的消息

那么对于所有t ≥ t₀ + τd，PC2满足ε ≈ d(2κτ + ξ)，其中近似假设μ + ξ ≪ τ。

这个定理给出了物理时钟同步精度的上界，并指出了时钟同步所需的时间。
