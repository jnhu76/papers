摘要

Bloom过滤器是一种简单的空间高效随机数据结构，用于表示集合以支持成员查询。Bloom过滤器允许假阳性，但当错误概率被控制得足够低时，空间节省通常超过这一缺点。

自1970年代以来，Bloom过滤器已被用于数据库应用，但直到近年来它们才在网络文献中流行起来。本文旨在调查Bloom过滤器在各种网络问题中的使用和修改方式，旨在为理解它们提供一个统一的数学和实践框架，并促进它们在未来应用中的使用。

本文首先描述Bloom过滤器背后的数学原理、历史和一些重要变体，然后考虑Bloom过滤器在网络相关应用中的四种类型：协作覆盖和对等网络、资源路由、数据包路由和测量。

引言

Bloom过滤器原理

Bloom过滤器原则：只要使用列表或集合，并且空间有限，如果假阳性的影响可以减轻，考虑使用Bloom过滤器。

统一这些不同应用的主题是Bloom过滤器提供了一种简洁的方式来表示项目集合或列表。在网络中有许多地方可能希望保留或发送列表，但完整列表需要太多空间。Bloom过滤器提供了一种可以显著减少空间的表示，代价是引入假阳性。如果假阳性不会引起显著问题，Bloom过滤器可以提供改进的性能。

数学基础

标准Bloom过滤器

表示n个元素集合S = {x₁, x₂, ..., xₙ}的Bloom过滤器由m位数组描述，初始全部设置为0。Bloom过滤器使用k个独立哈希函数h₁, ..., hₖ，范围是{1, ..., m}。

图 1: Bloom过滤器示例

过滤器开始是一个全0的数组。集合中的每个项目xᵢ被哈希k次，每次哈希产生一个位位置；这些位被设置为1。要检查元素y是否在集合中，将其哈希k次并检查相应的位。元素y₁不可能在集合中，因为在其中一个位找到了0。元素y₂要么在集合中，要么过滤器产生了假阳性。

技术要点
  • 初始状态所有位为0
  • 插入元素时设置多个哈希位置为1
  • 查询时检查所有哈希位置是否都为1
  • 假阳性可能发生但假阴性不会
原图图注: An example of a Bloom filter. The filter begins as an array of all 0s. Each item in the set xᵢ is hashed k times, with each hash yielding a bit location; these bits are set to 1. To check if an element y is in the set, hash it k times and check the corresponding bits. The element y₁ cannot be in the set, since a 0 is found at one of the bits. The element y₂ is either in the set or the filter has yielded a false positive.

假阳性概率分析

插入所有元素后，特定位仍为0的概率：

p′ = (1 - 1/m)kn ≈ e-kn/m = p

假阳性率：

f = (1 - p)k = (1 - e-kn/m)k

优化参数

最优哈希函数数量：

k = ln 2 · (m/n)

此时假阳性率：

f = (1/2)k ≈ (0.6185)m/n

网络应用

分布式缓存：Summary Cache

Fan等人描述了Summary Cache，它使用Bloom过滤器进行Web缓存共享。在他们的设置中，代理以下列方式协作：在缓存未命中时，代理尝试确定另一个代理缓存是否持有所需的Web页面；如果是，则向该代理发出请求，而不是尝试从Web获取该页面。

为了使这种方案有效，代理必须知道其他代理缓存的内容。在Summary Cache中，为了减少消息流量，代理不传输与其缓存确切内容对应的URL列表，而是定期广播表示其缓存内容的Bloom过滤器。

技术实现细节

  • 使用计数Bloom过滤器跟踪自己的缓存内容
  • 广播对应的标准0-1 Bloom过滤器给其他代理
  • 使用增量压缩和压缩Bloom过滤器进一步减少传输位数
  • 在开源Web代理缓存Squid中实现，称为缓存摘要

P2P/覆盖网络应用

中等规模P2P网络

对于数百个节点的中等规模和更健壮的对等系统，Bloom过滤器可能为在对等系统上定位对象提供有吸引力的分布式哈希表替代方案。

虽然在对等系统中保留每个其他节点存储的对象列表可能不可行，但保留每个其他节点的Bloom过滤器可能是可行的。例如，不是为每个对象使用64位标识符，Bloom过滤器可以为每个对象使用8或16位。

近似集合协调

Byers等人展示了Bloom过滤器在对等应用中可能有用的另一个领域。他们建议对等方可能希望解决以下类型的近似集合协调问题。

假设对等方A有一组项目SA，对等方B有一组项目SB。对等方B希望向对等方A发送一个简洁的数据结构，以便A可以开始向B发送B没有的项目，即SA - SB中的项目。

关键字搜索的集合交集

Reynolds和Vahdat以类似于数据库连接的方式使用Bloom过滤器，除了他们的目标是找到集合交集而不是集合差集。

对等方B可以向A发送表示SB的Bloom过滤器；然后对等方A发送根据过滤器似乎出现在SB中的SA元素。假阳性产生实际上不在SB中的SA元素，但如果需要，B可以然后确定这些元素以精确找到SA ∩ SB。

资源路由应用

基本路由协议

Czerwinski等人描述了资源发现服务架构的一部分的一般框架。

假设我们有一个根树形式的网络，节点持有资源。从树内部开始的资源请求朝向根节点。每个节点保留它持有或通过任何其子节点可达的统一资源列表，以及它和每个子节点的单独资源列表。

当节点收到资源请求时，它检查其统一列表以确保它有路由该请求到资源的方式。如果有，它然后检查单独列表以找到它是否持有资源或如何通过适当的子节点将请求路由到正确的节点；否则，它将请求进一步向上传递到根节点。

Rhea和Kubiatowicz利用第6.节基本协议中的思想为对等定位机制设计概率路由算法，与OceanStore项目结合。

目标是确保当请求的文件在系统中有附近的文件副本时，找到它并且请求沿着最短路径高效路由。

技术实现细节

环路检测机制

每个数据包在头部携带一个小型Bloom过滤器，用于跟踪访问过的节点集。每个节点有一个对应的掩码，可以在传递时OR到Bloom过滤器中；如果过滤器没有变化，可能存在环路。

队列管理算法

每个数据包到达基于源-目的地对的哈希递增计数Bloom过滤器的k个计数器。每个Bloom过滤器条目有一个关联值pᵢ，用于表示与该计数器关联的标记概率。

多播转发

不是为每个地址保留接口列表，每个接口可以有一个地址的Bloom过滤器。当具有多播地址的数据包到达一个接口时，检查所有其他接口的Bloom过滤器以查看具有该地址的数据包是否应该沿着该接口转发。

测量基础设施应用

记录重流

Estan和Varghese提出了Bloom过滤器在路由器内部流量测量问题中的优秀应用，让人想起随机公平蓝算法中使用的技术。

目标是轻松确定路由器中的重流。每个进入的数据包被哈希k次到Bloom过滤器中。与Bloom过滤器中的每个位置关联的是一个计数器，记录通过该位置与路由器关联的已通过数据包字节数。计数器按数据包中的字节数递增。

  • 保守更新只将计数器的值更改为反映与该流关联的最可能字节数
  • 减少假阳性率，特别是对于真实数据

IP追踪

Snoeren等人建议这种方法，并添加使用Bloom过滤器以减少需要存储的信息量，以总结看到的数据包集，作为其源路径隔离引擎(SPIE)的一部分。

此设置中的假阳性意味着路由器错误地将数据包识别为已看到。当尝试追溯数据包的反向路径时，假阳性将导致分支，给出多个可能路径。低假阳性率将保持分支小，因此可能路径的数量也小。

近期工作

自本调查的初始版本以来，Bloom过滤器已经有显著额外工作。我们简要强调一些相关的理论工作，我们预计这些工作将在未来的网络应用中有用。

光谱Bloom过滤器

Cohen和Matias引入光谱Bloom过滤器，它扩展基本Bloom过滤器构造以便它可以处理多集。他们工作的一个创新是使用第二个过滤器处理在过滤器中有唯一最小计数器的元素，以提高结果估计的准确性。

计数最小草图

Cormode和Muthukrishnan设计计数最小草图，Bloom过滤器上的变体，设计用于处理数据流上的几个问题，再次类似于Estan和Varghese的工作。他们能够提供理论保证，同时仅使用成对独立哈希函数；这是一个重大进步，因为成对独立哈希函数通常易于实现并且在实践中非常高效。

空间码Bloom过滤器

另一篇处理类似问题的论文引入空间码Bloom过滤器，它利用多个Bloom过滤器和最大似然估计以近似多集。推动此变体的目标是获得网络路由器上所有流的准确数据包计数估计，而不仅仅是重流。

Bloomier过滤器

Chazelle等人引入他们称为Bloomier过滤器的内容，它将Bloom过滤器扩展到处理集合S的每个元素与函数值关联的情况（来自离散和有限的值集）；宇宙的所有其他元素的函数值假定未定义。Bloomier过滤器为S中的任何元素提供适当的函数值，并为不在S中的任何元素返回对应于未定义的值，除了对于不在S中的元素，有一些假阳性的概率，Bloomier过滤器可能返回不正确的函数值。

结论

Bloom过滤器是一种简单的空间高效集合或列表表示，处理成员查询。正如我们在本调查中看到的，有许多网络问题需要这样的数据结构。特别是当空间是一个问题时，Bloom过滤器可能是保留显式列表的优秀替代方案。

使用Bloom过滤器的缺点是它允许假阳性。必须为每个特定应用仔细考虑它们的影响，以确定假阳性的影响是否可接受。这使我们回到：

Bloom过滤器原则：只要使用列表或集合，并且空间有限，如果假阳性的影响可以减轻，考虑使用Bloom过滤器。

似乎有足够的空间为特定应用开发Bloom过滤器的变体或扩展。例如，我们已经看到计数Bloom过滤器允许多集或通过插入和删除随时间变化的动态集的近似表示。Bloom过滤器现在开始受到算法社区的显著关注，虽然已经有了一些最近的结果，但很可能还有进一步的改进可以发现。

挑战与不足

技术局限性

假阳性问题

  • 假阳性是Bloom过滤器的固有特性
  • 在某些应用中假阳性可能导致性能下降或错误
  • 需要仔细权衡空间节省与假阳性率

动态操作限制

  • 标准Bloom过滤器不支持删除操作
  • 计数Bloom过滤器增加空间开销
  • 计数器溢出是潜在风险
