舍入误差

舍入误差是浮点计算的基本特征。由于实数无法完全用有限位表示，计算结果必须舍入以适应有限的表示形式。

灾难性抵消

当操作数本身含有舍入误差时，相近数的减法会导致大量精确数字消失。

例子: b² - 4ac

b² ≈ 11.2, 4ac ≈ 11.1

结果: 0.1 (误差70 ulps)

良性抵消

当操作数是精确已知时，相近数的减法误差很小。

例子: (x-y)(x+y) vs x²-y²

前者将灾难性抵消转为良性抵消

保护位的作用

没有保护位时:

x = 1.01×10¹
y = 0.99×10¹
x - y = 0.02×10¹ = 0.2 (错误!)

有保护位时:

x = 1.010×10¹
y = 0.993×10¹
x - y = 0.017×10¹ = 0.17 (正确!)

IEEE浮点标准

IEEE 754标准定义了二进制浮点算术的规范，包括格式、运算规则和异常处理。该标准的主要目标是提高浮点计算的可靠性和可移植性。

NaN和无穷大的示例

产生NaN的操作:

  • 0/0
  • ∞ + (-∞)
  • 0 × ∞
  • √x (当x<0时)

无穷大的操作:

  • 1/0 = ∞
  • 1/∞ = 0
  • ∞ + x = ∞ (x为有限数)
  • ∞ × ∞ = ∞

Kahan求和公式

用于高精度求和的算法:

S = X[1]
C = 0
for j = 2 to N {
  Y = X[j] - C
  T = S + Y
  C = (T - S) - Y
  S = T
}

该算法可以显著减少累加误差，误差界为O(Nε²)∑|x_j|，而不是简单求和的O(Nε)∑|x_j|。

编译器优化的陷阱

编译器不应随意对浮点表达式应用代数恒等式优化。例如，(x+y)+z 和 x+(y+z) 在浮点运算中可能产生不同结果，特别是当数值大小差异很大时。
