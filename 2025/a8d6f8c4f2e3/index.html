摘要

分层内存系统通常依赖访问频率("热度")来指导数据放置。然而，热数据并不总是性能关键的，这限制了基于热度的策略的有效性。

作者引入了摊销外核延迟(AOL)，这是一种新颖的指标，通过考虑内存访问延迟和内存级并行性(MLP)来精确捕捉内存访问的真实性能影响。

利用AOL，作者提出了两种强大的分层机制：Soar，一种基于性能贡献放置对象的配置文件引导分配策略；以及Atro，一种轻量级页面迁移调节策略，用于消除不必要的迁移。

Soar和Atro在多种工作负载上优于四种最先进的分层设计，性能提升高达12.4倍，而在少数情况下性能下降不超过3%。

背景与动机

分层内存架构

随着图处理和机器学习等内存密集型工作负载的需求增长，集成快速层(如DRAM)和慢速层(如CXL内存)的分层内存架构正在成为云数据中心的标准。

现有方法的局限性

现有的分层设计基于一个假设：频繁访问的("热")数据比冷数据更性能关键，应该驻留在快速层。因此，分层内存管理主要关注热度跟踪、内存分配和迁移策略，以高效地检测、分配和重新定位跨层的热数据。

关键问题

热数据并不总是性能关键的，并且可以驻留在慢速层而不会降低性能。在现代乱序CPU设计中，内存级并行性(MLP)等延迟缓解技术掩盖了内存访问的真实成本。

内存级并行性(MLP)的作用

MLP量化了内存控制器等待完成的未完成内存请求数量。它在减少CPU停顿时间方面起着关键作用。在高MLP访问模式下，CPU可以发出许多并发请求以最小化停顿。相反，序列化访问模式表现出低MLP并更直接地暴露延迟，导致更大的减速。

高MLP模式

  • 数组遍历等访问模式
  • CPU可以发出并发请求
  • 内存延迟被掩盖
  • 对慢速层访问不敏感

低MLP模式

  • 指针追逐等访问模式
  • 序列化访问
  • 内存延迟直接暴露
  • 对慢速层访问敏感

问题与挑战

现有分层系统的局限性

次优数据放置

现有的粗粒度分配策略优先为新分配的数据放置快速层，但在快速层压力下，性能关键数据经常被置换到慢速层，需要后期昂贵的迁移来纠正放置错误。

过度迁移开销

现有系统通常采用激进的迁移策略，通过频繁重新定位非关键页面产生大量开销。这种开销可能侵蚀或抵消分层带来的性能优势。

MLP建模的挑战

虽然MLP在架构社区中是一个成熟的概念，但其对分层内存管理的影响在很大程度上被忽视了。先前跨对象、页面和数据结构的分类工作通常通过粗略启发式或内存访问成本的间接指标隐式反映MLP的影响。然而，它们没有明确建模或量化MLP影响。

缺失的关键要素

缺少的是一个原则性的、准确的和MLP感知的性能指标，能够在在线和离线场景中实现更有效的、性能驱动的分层策略，并泛化到各种工作负载。

设计与实现

摊销外核延迟(AOL)

作者提出了摊销外核延迟(AOL)，这是一种新颖的性能指标，通过整合内存延迟和MLP来准确量化内存访问的性能影响。

技术细节：图2 - AOL性能预测
原图 2: AOL性能预测
论文原图
原图图注: AOL性能预测的准确性和有效性验证。
技术解释：
  • LLC-Stall准确性: LLC-Stall有效捕捉了工作负载在慢速层上的性能减速
  • 基础预测器: 基于LLC-Stall的基础预测器(P)倾向于高估高MLP工作负载的减速
  • MLP影响: 每个LLC-miss的平均stall在不同工作负载间变化可达4倍
  • AOL模型: 工作负载特定的修正因子K与AOL呈双曲线关系
  • 预测精度: 集成K的基础预测器显著提高了预测精度
  • 时间序列预测: AOL预测器泛化到细粒度时间序列预测
设计实现：
AOL定义:

AOL = Latency / MLP，其中Latency = A₁/A₃，MLP = A₂/A₁

基础预测器:

P = sLLC / c，其中sLLC是LLC停顿周期数，c是CPU周期数

修正因子:

K = f(AOL) = 1 / (a + b/AOL)，其中a和b是硬件相关常数

最终预测器:

S = P × K ≈ (sLLC / c) × (1 / (a + b/AOL))

性能分析：
测试配置:
  • 工作负载: 56个来自SPEC CPU 2017和GAPBS的工作负载
  • 硬件: Intel PMU计数器
  • 度量: LLC停顿周期、CPU周期、未完成请求
关键结果:
  • 预测精度: Pearson相关系数从0.869提高到0.951
  • 时间序列: 有效捕捉动态性能变化
  • 轻量测量: 仅需4个硬件计数器

Soar: 基于排名的静态对象分配

Soar是一种AOL驱动的配置文件引导内存分配策略，基于对象对工作负载性能的累积贡献进行排名。

原图 3: Soar分析流程
论文原图
原图图注: Soar跟踪多个信息流以基于性能影响推导对象排名。
技术解释：
  • 对象跟踪: 通过LD_PRELOAD拦截malloc()/free()和mmap()/mummap()，记录对象元数据
  • 内存访问跟踪: 使用Intel PEBS采样LLC缺失，记录时间戳和虚拟地址
  • 性能分析: 利用AOL预测估计内存访问性能影响
  • 流统一: 合并三个数据流构建全面的时间序列对象配置文件
  • 对象排名: 基于MLP和访问频率估计相对对象贡献
设计实现：
对象分组:

通过调用栈(call chain)分组对象，相同代码路径的对象视为同一类型

采样策略:

低采样率(如3000)足够，施加可忽略的开销并避免时序偏差

内存访问比率:

R = c / Σci，表示对象内存访问相对于总内存访问的权重

分配决策:

基于对象单位分数排名，将前N个对象放置在快速层

Atro: 基于AOL的自适应页面迁移

Atro是一种自适应分层编排策略，动态调节页面迁移强度。Atro利用AOL指标检测内存访问重叠高的时期，在这些时期慢速层访问的性能影响最小。

低AOL情况

低AOL表示高MLP，意味着内存延迟大部分被掩盖，潜在减速最小。Atro限制热页面检测、页面迁移或两者的速率，以减少不必要的开销。

高AOL情况

高AOL表示序列化、延迟敏感的内存访问，正确的页面迁移可以产生显著的性能增益。Atro通过启用更积极的页面检测和迁移来响应。

与现有系统的集成

Atro可以无缝集成到现有的分层系统中，如TPP、NBT、Nomad和Colloid，以增强其效率并减少开销。

系统 集成方式
Atro+TPP 基于离线分析阈值按比例限制页面迁移速率
Atro+NBT 在内存访问重叠显著时期限制设置为PAGE_NONE的页面数量
Atro+Nomad 调节从慢速层到快速层的页面迁移
Atro+Colloid 仅调节从慢速层到快速层的页面迁移

结论

作者提出了摊销外核延迟(AOL)，这是一种新颖的性能指标，通过整合内存访问延迟和内存级并行性(MLP)来准确量化内存访问的性能影响。

基于AOL，作者开发了两种强大的分层机制：

Soar

基于性能贡献的静态对象分配策略，实现近乎最优的数据放置

Atro

轻量级页面迁移调节策略，消除不必要的迁移开销

在多种工作负载上，Soar和Atro比四种最先进的分层设计性能提升高达12.4倍，而在少数情况下性能下降不超过3%。
